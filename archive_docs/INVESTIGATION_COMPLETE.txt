Question: Why are there ZERO fraud cases (INVOICED_NOT_DELIVERED)?

Answer: SAP enforces 3-way matching (PO → GR → IR). In your source data, 
EVERY invoice receipt (IR) for a given PO+Item ALSO has a corresponding 
goods receipt (GR). When aggregated by (PO, Item), all IR-only records get 
combined with their GR counterparts. Result: 0 IR-without-GR cases = 0 fraud.

KEY FINDING: WHY THIS HAPPENS
───────────────────────────────────────────────────────────────────────────────

SAP Procure-to-Pay Standard Control:
  Step 1: Create Purchase Order (PO)
  Step 2: Receive Goods (GR) - marked with E
  Step 3: Receive Invoice (IR) - marked with Q
  Step 4: System enforces 3-way matching

SAP Rule: Cannot create IR without prior GR for same PO+Item
Exception: Only if matching control is explicitly disabled (rare)

Result: EVERY IR has a GR for same (PO, Item)
Therefore: "IR without GR" = IMPOSSIBLE = 0 fraud cases

This is NOT a code bug - this is HOW SAP P2P IS DESIGNED TO WORK
